# Security Policy

Please report security-sensitive issues privately instead of opening a public issue.

If no private advisory channel is available in GitHub, open a minimal public issue asking for a private contact path, without including exploit details, credentials, private SQL, or customer data.

## Public issue safety

When filing public issues, remove:

- passwords and tokens
- hostnames and private IP addresses
- connection strings
- customer data
- proprietary SQL and schema details
- screenshots containing confidential information
